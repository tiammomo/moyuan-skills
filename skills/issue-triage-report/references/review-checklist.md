# Review Checklist

## Before Circulation

Check these items before sending the report:

- urgent issues are still actually urgent
- resolved issues are not still presented as active blockers
- decision items are truly unanswered decisions
- owner names are current
- customer impact wording is appropriate for the audience
- placeholder text or empty sections are not left behind

## Sensitive Content Check

Be especially careful with:

- customer names
- security details
- internal-only incident context

If the audience is broad, remove or generalize them.

## Final Review

Use at least one of these before circulation:

- run the bundled linter
- do a human editorial pass
