# 初学者学习路径

这条路径面向第一次接触 skill 和 harness thinking 的读者。

## 目标

学会三件事：

- 看懂这个仓库每一层在干什么
- 自己做出一个最小可用 skill
- 知道 skill 为什么不能写成一篇越来越长的总说明书

## 最推荐的起点

如果你是新人，不要直接从最长的文档读起。

建议先从 [14-first-hour-onboarding.md](../onboarding/14-first-hour-onboarding.md) 开始，再回到这条路径。

如果你在读到一半时发现自己卡在术语上，再插入：

- [15-newcomer-faq.md](../onboarding/15-newcomer-faq.md)

## 第一小时的建议产出

第一次接触这个仓库时，只要求自己先完成这三件事：

- 说清这个仓库的三层结构
- 写出一个 skill 名称和一条 `description`
- 跑通至少一条 checker 或 eval 命令

做到这里，就已经比“把文档看很多但还没动手”更有价值。

## 学完后应该能做到

- 分清 frontmatter、`SKILL.md`、`references/`、`scripts/`、`assets/`
- 看懂一个 router 是怎么把问题收窄的
- 做出一份带最小 checker 的 skill 草稿

## 新人最容易犯的错

- 一开始就想把 skill 写成万能工具
- 先写很长的正文，再回头补触发句
- 还没建立最小心智模型，就直接跳进 harness 细节
- 看到术语先硬记，不先回头确认它在仓库里到底承担什么角色
