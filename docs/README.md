## 文档治理规则

- 根目录文档只放当前有效的长期说明，不保留“下一轮临时计划”。
- 某个主题如果已经被主线 roadmap 覆盖，应合并回主线文档并删除重复说明。
- 一次性排障、局部迭代和会过期的 TODO，优先放在 issue、PR 或任务系统里，不再新增到 `docs/` 根目录。
